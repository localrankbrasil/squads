# Checklist Tecnico — Landing Page Local Rank

## Preview GitHub

- [ ] `output/github-preview/index.html` abre sem build step.
- [ ] Nao ha links quebrados essenciais.
- [ ] Nao ha placeholders visiveis.
- [ ] Shortcode Fluent Forms esta presente.
- [ ] HTML tem `lang="pt-BR"`.
- [ ] Viewport meta esta presente.
- [ ] Fontes Merriweather e Roboto carregam.
- [ ] CSS usa `box-sizing: border-box`.

## Responsivo

- [ ] Desktop sem overflow horizontal.
- [ ] Mobile sem overflow horizontal.
- [ ] Grids colapsam para uma coluna.
- [ ] Texto nao sobrepoe CTA, cards ou formulario.
- [ ] Botoes continuam legiveis e clicaveis.
- [ ] Padding mobile minimo de 20px.

## Acessibilidade

- [ ] Contraste adequado para textos principais.
- [ ] Botoes e links tem texto claro.
- [ ] Estados `:focus-visible` presentes nos CTAs.
- [ ] Headings seguem hierarquia logica.
- [ ] Imagens, quando houver, tem `alt` util ou decorativo vazio.

## Branding

- [ ] Cores principais pertencem ao brand kit.
- [ ] Verde nao domina a pagina como fundo principal.
- [ ] Tipografia segue Merriweather/Roboto.
- [ ] Visual nao parece template SaaS generico.
- [ ] Tom visual permanece consultivo e profissional.

## Fidelidade a Copy

- [ ] Headline e promessa preservam a intencao original.
- [ ] Nenhum dado ou resultado foi inventado.
- [ ] Nenhuma nova oferta foi adicionada.
- [ ] CTAs respeitam a copy recebida.
- [ ] FAQ nao cria objecoes novas sem base no texto.

## Handoff Elementor

- [ ] Todas as secoes do preview estao no `section-map.md`.
- [ ] `elementor-build-guide.md` tem ordem de montagem.
- [ ] `custom-css.css` usa classes com prefixo `lr-landing-`.
- [ ] `fluent-forms-placement.md` informa onde inserir o shortcode.
- [ ] Regras mobile estao especificadas.
