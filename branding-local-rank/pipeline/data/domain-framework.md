# Framework de Auditoria de Marca — Local Rank Brasil

## Metodologia de Brand Consistency Audit

### Fase 1: Recebimento e Classificação do Material
1. Identificar o tipo de material: carrossel Instagram, post, story, email, copy de site, ad copy, template Canva
2. Identificar a plataforma de destino: Instagram, email, WhatsApp, site, Google Ads
3. Identificar a persona/público alvo do material (Caroline, Andréia, Marcus, Leandro ou público geral)
4. Classificar o estágio do funil: topo (awareness), meio (nutrição) ou fundo (conversão)

### Fase 2: Auditoria Visual (se material tem elementos visuais)
1. **Cores**: listar todas as cores usadas e verificar contra a paleta aprovada
2. **Tipografia**: verificar fontes (Merriweather/Roboto) e tamanhos mínimos por plataforma
3. **Logo**: verificar presença, variação correta para o fundo, área de respiro
4. **Hierarquia**: verificar leitura clara (título > subtítulo > corpo)
5. **Contraste**: verificar legibilidade do texto sobre o fundo (WCAG AA mínimo)
6. **Dimensões**: verificar proporção e resolução para a plataforma

### Fase 3: Auditoria Verbal (copy, texto, mensagem)
1. **Tom de voz**: verificar alinhamento com os 4 pilares (educativo, empático, direto, acessível)
2. **Vocabulário**: identificar termos proibidos (jargões técnicos sem contexto, promessas irreais)
3. **Mensagem central**: verificar se reforça o posicionamento Google-first
4. **Adequação à persona**: verificar se o tom é adequado para o público identificado
5. **CTA**: verificar se o call-to-action é claro, específico e não-agressivo

### Fase 4: Pontuação por Critério
1. Pontuar cada critério de 1-10 com justificativa específica
2. Identificar critérios críticos com nota < 4 (rejeição automática)
3. Calcular score médio
4. Determinar veredito: APROVADO, APROVADO CONDICIONAL ou REPROVADO

### Fase 5: Recomendações
1. Para cada critério reprovado: especificar o problema, o local exato e a correção necessária
2. Para cada critério condicional: listar como "Required change"
3. Para melhorias não-críticas: listar como "Suggestion (non-blocking)"
4. Se REPROVADO: definir "Path to Approval" com lista ordenada de mudanças obrigatórias

## Framework de Brief de Design

### Quando Gerar um Brief
- Material reprovado na auditoria (correção necessária)
- Material novo a ser criado do zero
- Template a ser construído no Canva

### Estrutura do Brief
1. **Sistema de Design**: cores exatas (hex), fontes (família + peso + tamanho), espaçamentos, grid
2. **Slide-a-slide / Elemento-a-elemento**: descrição de cada componente visual com especificações
3. **Textos aprovados**: copy final para cada elemento de texto
4. **Referências visuais**: links para carrosséis referência da pasta Canva
5. **Canva template**: especificar qual template base usar e onde encontrar no brand kit

### Critérios de Qualidade do Brief
- Toda cor é especificada em hex
- Toda fonte tem família + peso + tamanho + cor
- Todo elemento tem posição/alinhamento especificado
- Sem ambiguidade: quem receber o brief consegue executar no Canva sem perguntar nada
